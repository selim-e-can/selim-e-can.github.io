Modular Gas Sensor Shell

I designed and 3D-printed a modular gas sensor shell (E-nose) that stores a Raspberry Pi Zero 2W and 17 gas sensors to identify chemical signatures (volatile organic compounds, temperature, humidity) humans perceive as "smells".

Design Specifications

  • 1. There must be uninterrupted gas flow with the gas sensors properly oriented for measurements
  • 2. The product must be modular and lightweight (less than 1kg).
  • 3. The sensors, the wiring and the motherboard should be protected by a modular and easy-to-remove gas sensor shell.

Design Considerations

Figure 1: The gas sensor assembled.

Figure 2: The parts on the rightside of the image are "chambers". They are modular and hold the PCB, Raspberry Pi Zero 2W, and all of the sensors. The resulting stacked chamber structure is then placed inside the "shell structures" on the leftside of the image. The resulting stacked shell structure, which is also modular, protects the mentioned components and stores necessary wiring. The legs at the bottom hold up the gas sensor, so that gas can flow from the bottom of the sensor and exit from the top.

The Modular Gas Sensor Shell stores 17 sensors:
  • DFRobot CO2 Sensor V1.2
  • DHT22 Temperature and Humidity Sensor
  • DFRobot Ozone Sensor (0-10ppm) v1.0 (I2C)
  • DFRobot Gravity Formaldehyde (HCHO) Sensor (DAC)
  • SGP40 VOC (Volatile Organic Compound) Sensor (I2C)
  • Figaro TGS2611 Air Contaminants Sensor with drive board module
  • CJMCU-1100 MS-1100 VOC Gas Sensor Module Formaldehyde Benzene Gas Induction
  • MH-Z14 NDIR CO2 Infrared Sensor Module MH-Z14A Carbon Dioxide Sensor for CO2 (0-5000ppm, Serial Spot PWM)
  • MQ-2, MQ-3, MQ-4, MQ-5, MQ-6, MQ-7, MQ-8, MQ-9, MQ-135 Smoke Detection liquefied Methane Gas Sensor Module

The gas sensor shell houses all components required for data collection and processing: the gas sensors mentioned above, Raspberry Pi Zero 2W, a custom PCB, and an LCD display. The hardware is constructed from ABS plastic, ensuring durability while keeping the overall system lightweight at 830.70 grams. The shell features a modular design with "lips" on the top of both the chamber and shell structures, allowing easy customization and the use of any number of gas sensors (inspired by the concept of linked lists). The topmost node in the structure houses a custom PCB and a Raspberry Pi Zero 2W which handles all the data processing instead of collecting data (inpspired by the concept of a dummy node). The shell corresponding to this node houses an LCD display that can be used to visualize the object(s) associated with the measured chemical signature or display relevant gas concentrations. The shell's top and bottom caps include rectangular cutouts designed to accommodate fans, which can be installed to enhance gas flow if necessary.

The system's core functionality relies on multiple gas sensors that collect data to form a chemical signature associated with the underlying process (i.e. cooking food, perfume, smoke, etc...). The chambers are equipped with side holes where the gas sensors are securely mounted, ensuring optimal alignment with the gas flow.



Figure 3: The PCB stores a Raspberry Pi Zero 2W, 2 MCP3008 ADC Converters which provide connections to 14 analog gas sensors, two I2C connections, and a connection for the temperature and humidty sensor.

The PCB, shown in Figure 2, is designed with a range of functional components to support the system's operations. On the left side, general-purpose buttons are included, with one set designated for starting and stopping data collection. The upper-right corner of the PCB features a power supply auxiliary connector, allowing the use of an external power source to power the sensors, as the Raspberry Pi Zero 2W cannot provide sufficient power for all sensors. At the bottom, the PCB includes connections for ADC converters and gas sensors that communicate using the I2C protocol. Additionally, the bottom-right corner houses optional transistors and pins designed to power a fan, which can be activated if gas flow needs to be enhanced. Cutout holes and rectangular slots on the top face of the gas sensor shell are precisely aligned to accommodate the buttons and power supply auxiliary connector, ensuring a seamless and functional design.
