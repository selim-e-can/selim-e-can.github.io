Modular Gas Sensor Shell

I designed a gas sensor shell to 1) store an array of sensors oriented correctly relative to the gas flow 2) have enough space to accomadate wiring, a PCB and a headless Raspberry Pi 3) be modular to add/remove sensors if needed.

Design Specifications

  • 1. The PCB and Raspberry Pi should not block gas flow, gas sensors must be properly oriented, and the product must be lightweight (less than 1kg).
  • 2. The product must be modular and store a PCB, Raspberry Pi Zero 2W, and wirings necessary for all the sensors.
  • 3. The gas sensor shell can be independently removed without removing the PCB, Raspberry Pi Zero 2W, the wires, and the sensors.

Current Design Considerations

Figure 1: The parts on the rightside are "chambers". They are modular and hold the PCB, Raspberry Pi Zero 2W, and all of the sensors. The chamber structure is then placed inside the "shell structures" on the leftside which are also modular. The shell structure protects the mentioned components and stores necessary wiring. The legs at the bottom hold up the gas sensor.

Currently, the Modular Gas Sensor Shell stores 17 sensors:
  • DFRobot CO2 Sensor V1.2
  • DHT22 Temperature and Humidity Sensor
  • DFRobot Ozone Sensor (0-10ppm) v1.0 (I2C)
  • DFRobot Gravity Formaldehyde (HCHO) Sensor (DAC)
  • SGP40 VOC (Volatile Organic Compound) Sensor (I2C)
  • Figaro TGS2611 Air Contaminants Sensor with drive board module
  • CJMCU-1100 MS-1100 VOC Gas Sensor Module Formaldehyde Benzene Gas Induction
  • MH-Z14 NDIR CO2 Infrared Sensor Module MH-Z14A Carbon Dioxide Sensor for CO2 (0-5000ppm, Serial Spot PWM)
  • MQ-2, MQ-3, MQ-4, MQ-5, MQ-6, MQ-7, MQ-8, MQ-9, MQ-135 Smoke Detection liquefied Methane Gas Sensor Module
The data gathered from the sensors listed above during the cooking process will constitute a chemical signature which will be used to analyze the cumulative gas concentration with respect to time. The gas sensor shell stores these sensors, and the hardware which extracts and processes data which includes a custom PCB and a Raspberry Pi Zero 2W. When designing the gas sensor shell, much consideration was given to requirement 3. Both the chamber and shell structures have "lips" on top of them, making the structure modular - allowing the use of any amount of gas sensors. One challenge I faced was designing a gas sensor shell that could be removed without removing the Raspberry Pi Zero 2W and the PCB (these components were stored within the shell in the previous design made by my graduate student mentor). To overcome this, I used the concept of a dummy node from my data structures and algorithms class: I made a special chamber sturcture that stores the PCB and the Raspberry Pi instead of gas sensors- satisfying requirements 2 and 3. Furthermore, beacuse the PCB and Raspberry Pi are on the side of the special chamber they do not interfere with the gas flow. The rest of the chambers have various holes on the side where gas sensors will fit and face towards the gas flow. In addition to all of this, There is a LCD display attached to the topmost gas sensor shell structure which allows the user to view any given gas concentration. There are also rectangular cutouts on the top and bottom caps of the gas sensor shell which is designed to store fans if the gas flow is not fast enough. Finally, All parts are mode from ABS plastic and have a total mass of 830.70 grams - satisfying requirement 1.

Figure 2: The PCB stores a Raspberry Pi Zero 2W, 2 MCP3008 ADC Converters which provide connections to 14 analog gas sensors, two I2C connections, and a connection for the temperature and humidty sensor.

The PCB is displayed on figure 2. On the left side of the PCB are functional buttons that are for general purpose use. One set of buttons will be used for triggering data collection and stopping data collection. On the upper rightside of the PCB is a power supply aux for an external power source which will power the sensors because the Raspberry Pi Zero cannot power all the sensors by itself. The bottom part of the PCB are connections for the ADC converters and gas sensors that use the I2C protocol. The bottom rightside of the PCB has optional transistor and pins that will be used to power a fan if gas flow is not sufficient. THe cut out holes and rectangles on the top face of the gas sensor shell are made to store the buttons and the power supply aux mentioned here.

Ending Remarks

A significant portion of this project was configuring the Raspberry Pi Zero 2W system by installing Raspbian Lite, setting up an SSH connection and writing the python script that will be used for data collection. Modular Gas Sensor Shell CAD files, links to documentation and python files for each sensor, necessary installations, and the python file that will be used to collect data can be found here. I have already 3D printed the Modular Gas Sensor as of now and will keep updating this webpage. Testing will begin according to the YORI Team schedule and the Modular Gas Sensor will be used by the YORI (RoMeLa) team cooking robot.
